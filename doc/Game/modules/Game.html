Module Game

Game represents the main engine that handles GUI, key presses and releases and game states.

Game states are something like scenes with their own render, update, init, onActivate and onDeactivate functions and their own GUI Containers. When there is a state change, Game changes the GUI to the one of the state. Also Game has some containers that hold sprites, sounds and fonts, so that you will not have the same asset multiple times in memory.

Info:

  • Author: Steve Wolligandt

State managing

init (initialState) initializes the engine
render () renders current game state, gui and textoutput
update (dt) updates current game state, gui and keyhandling
changeState (state) changes current game state and calls some callbacks

Memory Management

getFont (path, size) loads fonts, if font is already loaded it returns a reference to this
getSound (path, type) loads sounds, if sound is already loaded it returns a reference to this
tidyUpSpritePool () clears the sprite pool
tidyUpSoundPool () clears the sound pool
tidyUpFontPool () clears the font pool
