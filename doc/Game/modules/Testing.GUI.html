Module Testing.GUI

GUI Test creates two containers with buttons and labels, buttons change containers Our Game Engine does everything to update, render and handle key inputs for the gui, so in this example there will not be any function calls depending those issues

Info:

  • Author: Steve Wolligandt

Testing.GUI.init () creates the gui
Testing.GUI.render () renders gui and some background
Testing.GUI.fillContainer1 () shows how to create container and how to add widgets to it
Testing.GUI.fillContainer2 () shows how to create container and how to add widgets to it
Testing.GUI.renderBackground () renders background scene
Testing.GUI.createBackground () creates a scene for some background rendering
Testing.GUI.activeKeyBinding Keybindings of game state Testing.GUI
