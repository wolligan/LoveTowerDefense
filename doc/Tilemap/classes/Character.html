Class Character

Basic Character that can interact on the tilemap and can also be an AI.

Info:

  • Author: Steve Wolligandt

Functions

Tilemap.Character:new (apparentScene, x, y) Constructor
Tilemap.Character:render () Renders the Character (This should be overwritten in your derived class)
Tilemap.Character:update (dt) Renders the Character (This should be overwritten in your derived class)

Character Movement

Tilemap.Character:moveUp (dt) Moves the Character up, also regards the speed under the tile the character is located at
Tilemap.Character:moveLeft (dt) Moves the Character left, also regards the speed under the tile the character is located at
Tilemap.Character:moveDown (dt) Moves the Character down, also regards the speed under the tile the character is located at
Tilemap.Character:moveRight (dt) Moves the Character right, also regards the speed under the tile the character is located at

Collision Detection

Tilemap.Character:checkObstacleX () checks if the Character is caught in an obstacle in x-coordinate and resets to the old x-coordinate if thats the case
Tilemap.Character:checkObstacleY () checks if the Character is caught in an obstacle in y-coordinate and resets to the old x-coordinate if thats the case
Tilemap.Character:getTileIndex () returns the tile index the character is standing on
Tilemap.Character:getTileCoordinate () returns the tile coordinates the character is standing on
Tilemap.Character:keepCharInMap () assert that the character stays in range of the map

AI functions

Tilemap.Character:AI_think (dt) Think function gets repeatedly called from update.
Tilemap.Character:AI_walkToGoal (dt) Call this in your think method to let the character move to its goal if defined
Tilemap.Character:AI_calculatePathToGoal (goalX, goalY) calculate a path to a goal
